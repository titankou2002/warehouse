# 倉庫棧板判讀規則

這份文件記錄「倉庫全集」目前採用的棧板判讀方式，供後續接手與調整程式時參考。

## 1. 分頁與區塊概念

- 每個分頁會先有 `第一排 / 第二排 / 第三排...`
- 每一排底下是該排的棧板資料
- `A 欄` 不是主要給人看的結構依據，不應作為主要判斷來源

## 2. 單一棧板的資料形式

- 在 `第一排 / 第二排 / 第三排...` 下面：
  - 第 1 行一定是 `編號`
  - 第 2 行一定是 `批號`
  - `編號` 右邊是 `箱數`
  - `批號` 右邊是 `片數`

## 3. 白色區塊規則

- **白色 = 沒上色 = 1 個編號就是 1 板**
- 白色區塊不拆多板
- 看到白色時，預設就是「一行一板」

## 4. 彩色區塊規則

- **紅 / 黃 / 綠** 才是需要辨識物理棧板邊界的區塊
- 同色連續段視為同一個物理棧板區塊
- 一個彩色物理棧板可以包含多組 `編號 / 批號 / 箱數 / 片數`

## 5. 邊界與樣式

- 目前主要邊界方案以 **Bold 標記** 為主
- 每個物理棧板的第一個品號 `SKU` 行設為 `bold`
- 舊資料若沒有 `bold`，則 fallback 到同色塊偵測
- 不插空行 / 分隔行到試算表
- 不增加額外欄位
- 測試區若要提升人類閱讀性，可用 **深藍色上邊框** 做視覺分隔，不改資料內容

## 6. 判斷總結

- 白色：`1 行 = 1 板`
- 彩色：`同色連續段 = 1 個物理棧板`
- `第一行 = 編號`
- `第二行 = 批號`
- `編號右邊 = 箱數`
- `批號右邊 = 片數`
